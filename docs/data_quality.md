# Data Quality Notes

Generated by `scripts/data-registry-agent.mjs`.

## MVP Readiness

- Required datasets listed: 11
- Optional datasets listed: 4
- Blocked datasets listed: 0

The current data matrix is sufficient for a deterministic pre-feasibility ranking prototype, provided outputs are framed as screening signals and not final project decisions.

## Major Gaps Not Solved By Open Data

- Land tenure and user rights.
- Community willingness.
- Grazing pressure and seasonal land use.
- Local conflict, safety, and access constraints.
- Measured biomass or verified carbon stock.
- Local species survival rates.
- Nursery capacity and implementation cost.
- Field biodiversity baselines.

## Dataset Limitations

- chirps_rainfall
  - Resolution is coarse relative to 5-10km candidate sites.
  - Do not over-weight rainfall compared with field hydrology.
- cifor_icraf_species_atlas
  - Use as context for suggested species, not as a deterministic field prescription.
  - Local nursery availability and community preference still require validation.
- esa_worldcover
  - Categorical classes need mapping into project-specific restoration classes.
  - Classification errors are possible at local scale.
- ethiopia_restoration_atlas
  - Layer download/API availability must be verified before automated extraction.
  - Atlas classes should be treated as context or weak labels, not ground truth.
- gfw_umd_forest_change
  - Tree-cover loss is not the same as carbon loss or degradation severity.
  - Threshold choices affect forest/non-forest interpretation.
- ghsl_built_up_settlement
  - Built-up settlement proxies do not capture informal access or land tenure.
  - Layer choice should be documented.
- hdx_ocha_eth_admin_boundaries
  - Newer South Ethiopia and Southwest Ethiopia boundaries may require name cleanup or crosswalks from older SNNPR-era data.
- landsat_collection2_sr
  - Trend windows need consistent sensor and cloud masking choices.
  - Use as trend context, not precise biomass measurement.
- osm_geofabrik_ethiopia
  - Road completeness varies by area.
  - Distance to road is a feasibility proxy, not a full cost model.
- sentinel1_grd
  - Requires preprocessing choices for orbit direction, polarization, speckle, and terrain effects.
  - Use only if time allows.
- sentinel2_sr_harmonized
  - Cloud and shadow masking required.
  - Recent seasonal windows must be selected consistently.
- soilgrids
  - Modeled global product, coarse for project-level site decisions.
  - SOC proxy must not be presented as measured project carbon.
- srtm_dem
  - Slope is an erosion proxy, not a direct erosion measurement.
  - Micro-topography and local drainage are not captured.
- wdpa_protected_areas
  - Boundary accuracy and designation status should be checked before operational decisions.
  - Overlap is a safeguard flag, not an automatic no-go decision.
- worldpop_population
  - Population pressure is a proxy for livelihood relevance and human pressure, not community support.
  - Settlement patterns can be misrepresented in sparse rural areas.

## Required Product Guardrails

- Treat carbon outputs as potential or pre-feasibility signals unless measured field carbon exists.
- Include field-validation flags when social feasibility is inferred only from population, roads, or settlement proximity.
- Do not use protected-area overlap as the only safeguard check.
- Keep every numeric score traceable to a source dataset or mark it as mock/demo.
